The `DataGrid` is used to represent data that have no clear graphical distinction between items. Under the hood, it's built with `HTML table`.

The `DataGrid` component shows provided data in the table. Moreover, it includes pagination and additional buttons in the action section (above table header) with a popup to show/hide table columns.

The `DataGrid` is wrapped inside `Card` component. The table is in the middle, the action section is above it, and the pagination is at the bottom.
Table shows data based on provided data list and template. The template describes how to render each row.
The last column of a table is used to put row action, such as `ContextMenu` component.
The table header has support for single- & multi-sorting.

# Table

The heart of `DataGrid` component is the `HTML table`. The able header is generated based on provided `headers` props.
Table body is generated based on provided `data` and template.

```jsx
<DataGrid
  headers={[
    { name: 'name', headline: 'Name' },
    { name: 'created', headline: 'Created' },
    { name: 'id', headline: 'Identifier' },
    { name: 'type', headline: 'Type' },
    { name: 'enabled', headline: 'Enabled' },
  ]}
  data={[...]}
>
  {({ item }) => (
    <DataGridRow key={item.id}>
      <DataGridCell>{item.name}</DataGridCell>
      <DataGridCell>{item.created.toLocaleDateString()}</DataGridCell>
      <DataGridCell>{item.id}</DataGridCell>
      <DataGridCell>{item.type}</DataGridCell>
      <DataGridCell>...</DataGridCell>
      <DataGridCell>
        <ContextMenu ... />
      </DataGridCell>
    </DataGridRow>
  )}
</DataGrid>
```

## Table header

The `headers` property is a list of header objects. These objects contain column name (to be passed in sorting callback), a headline that is used to render header cell content,
and optionally `hidden` & `disableSorting` properties that indicate the visibility of the column (in the table header and in the table body) and disable sorting.

In the below example `Created` & `Identifier` columns are hidden. Sorting is possible only by `Name` column.

```jsx
<DataGrid
  headers={[
    { name: 'name', headline: 'Name' },
    { name: 'created', headline: 'Created', hidden: true },
    { name: 'id', headline: 'Identifier', hidden: true },
    { name: 'type', headline: 'Type', disableSorting: true },
    { name: 'enabled', headline: 'Enabled', disableSorting: true },
  ]}
  data={[...]}
  onSort={...}
  initialSort={[ { name: 'name', direction: 'ASC' } ]}
/>
```

### Sorting

The sorting indicators (icons) are not rendered until `onSort` callback function is set.
By clicking header cells `onSort` callback is called with parameters indicating the sorting column name and direction.
Each column has a triple-option state. Clicking once on the header cell sets the sorting direction to `ascending`, the second click sets it to `descending` and the third one turns off sorting.
Clicking on the other header cell sets the sorting direction to `ascending` for the current column - the previous one is forgotten.

By default, single-column sorting is enabled. To enable multi-sorting `enableMultiSorting` property should be set.
By enabling multi-sorting `onSort` callback takes an array of column names & directions pairs. It means that each column has its independent sorting state.
Clicking on different header cells doesn't make previous once to be forgotten, but moves those to be later in array.

_NOTE: The order of sorting pairs reflects the sort priority._

`initialSort` property consumes the initial settings for the sorting feature. To support single and multi sorting, it consumes an array of column names and directions pairs.

## Table body

Rows are generated based on provided template.
The template should wrap content in `DataGridRow` component. Each cell should be wrapped with `DataGridCell` component.
The last cell of each row is reserved for item actions. `ContextMenu` component is supposed to handle all actions that can be done within a row.

```jsx
<DataGrid
  headers={[
    { name: 'index', headline: '#' },
    { name: 'name', headline: 'Name' },
    { name: 'number', headline: 'Random number' },
  ]}
  data={[...]}
>
  {({ item, index }) => (
    <DataGridRow key={item.id}>
      <DataGridCell>{index + 1}</DataGridCell>
      <DataGridCell>{item.name}</DataGridCell>
      <DataGridCell>{Math.random()}</DataGridCell>
      <DataGridCell>
        <ContextMenu ... />
    </DataGridRow>
  )}
</DataGrid>
```

### Disabling context menu column

To disable context menu column set `disableContextMenuColumn` property.

### Skeleton loading animation

To show loading animation, set the `isLoading` property to `true`.
It renders 10 rows with skeleton loading animation in all defined cells.

```jsx
<DataGrid isLoading />
```

### Showing empty table message

To show an empty table message, set the `emptyLabel` property with the text to display.
Text will be displayed only if the `data` property is empty.

```jsx
<DataGrid emptyLabel="There are no vegetables within the current selection" data={undefined} />
```

# Action section

It's possible to show the `Add item`, `Columns` and `Search` buttons.

Clicking the `Columns` button opens the `Show columns` popup with a list of available columns whch can be togged.
However, the developer must handle the `Add item` and `Search` button actions.

Each button can be customized by providing props as in bellow example.

```jsx
<DataGrid
  actions={{
    enableAddBtn: true,
    enableColumnsBtn: true,
    enableSearchBtn: true,
    searchBtnProps: { title: 'Zoeken', children: 'Zoeken', onClick },
  }}
/>
```
