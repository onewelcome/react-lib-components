The typography component helps present text content. It has styling defined for all kinds of uses. Text can be rendered within all valid HTML tags, regardless of the proper styling setting that should be used (e.g., text in div can be styled as an H1 element).

## Font

Typography was designed to support any font you like. This can be changed by the configuration provided to the `BaseStyling` component. By default `Roboto` font is set.
`Roboto` is not automatically loaded. You are responsible for including font in your application.

<table>
  <thead>
    <tr>
      <th>
        NOTE: `code block` is the only exception that uses the font `Roboto Mono`. You only have to
        include it in your application if you use it.
      </th>
    </tr>
  </thead>
</table>

## Google fonts

You can take the HTML snippet beneath and put it in your application.

```html
<link
  href="https://fonts.googleapis.com/css2?family=Roboto:ital,wght@0,400;0,500;0,700;1,400;1,500;1,700&display=swap"
  rel="stylesheet"
/>
<link href="https://fonts.googleapis.com/css2?family=Roboto+Mono&display=swap" rel="stylesheet" />
```

## Install with npm

Alternatively, you can install font as a dependency using npm to get the benefits of self-hosting fonts. Read [more](https://www.npmjs.com/package/@fontsource/roboto).

```sh
npm i @fontsource/roboto @fontsource/roboto-mono
```

## Using spacing

By default all `Typography` elements have a `margin-bottom` property set to `1rem`. This can be changed by setting `spacing` property.
